Coffer

Description

Coffer is a very versatile blob storage engine written in Erlang.

It allows you to store blobs (set of bytes) in a given container.

Features

  • Multi-backend (FS, Distributed FS, S3...)
  • Simple REST API: GET,PUT, POST, DELETE
  • Synchronisation between containers
  • All blobs are uniquely identitified (content-hash)
  • Works with mobile & embedded devices
  • Partials upload & downloads supported
  • RESTFUL, resources visible on HTTP

Coffer Client

Description

Coffer is a small HTTP client written in erlang, able to do most operations on Coffer such as:

  • Listing blobs
  • adding, fetching blobs
  • asking if a given blob exists
  • and more to come

RCouch

Description

RCouch is a static distribution of couchdb using rebar. It is an extraction of the refuge code to ease the distribution of couchdb.

Based on the couchdb code it is tagged from time to time for stability purpose. At some point some supported branch may appear.

RCouch differs from refuge by only focusing on Apache CouchDB. It won't contain any of the refuge features like P2P nodes handling, ... . Internally we are using it to test couchdb and as an example on how to embed couchdb in your own applications.

RCouch is brought to you by the Refuge project.

Features

  • Apache CouchDB based
  • Geocouch integrated
  • rebar
  • relocatable
  • Fully opensource. All the sources are on refuge GIT repository (http://github.com/refuge) under Apache License 2.
